SUMMARY OF CHANGES

Changes made are reflective of the stand up of the Global Logistics Support Center (GLSC).

(SHAWAFB) This document has been substantially revised and must be completely reviewed.
Major changes include delegated responsibilities and processing authorization(s).
2.1. Overview. Most of the duties outlined in this chapter are performed by 735 SCMG (735
SCMG/LGPSC) Computer Operations. 735 SCMG Computer Operations is responsible for
processing the existing HQ ACC Command programs for all host accounts. Base Computer
Operations will develop local Query Language Processor (QLP) and Supply User Report
Generator (SURGE) programs as deemed necessary. See Attachment 2P-1, HQ ACC Supply
Programs, for a listing of HQ ACC Supply programs and their processing frequency.
2.6.1. 735 SCMG Computer Operations must be contacted at DSN: 575-0433/0435 by base
level Computer Operations prior to contacting Standard Systems Group.
2.17.1.5. Base and 735 SCMG Computer Operations will appoint a Terminal Area Security
Officer (TASO). (TASO responsibilities are outlined in Unisys STIG, V5R1).
2.21. User Access Validation. In addition, SATS user-IDS and TRIC validation will also be
conducted quarterly using the same criteria.
2.37. Designation of Test Bases. 735 SCMG/LGPS Management and Systems is responsible for
management and control of all actions required to support designated Standard Base Supply
System (SBSS) Qualification Test and Evaluation (QT&E) test bases as outlined in Section F.
2.38. Designation of Test Monitors. 735 SCMG/LGPS will appoint two test monitors (one
primary and one alternate) in the Management and Systems Flight. The QT&E base will appoint
two test monitors (one primary and one alternate) in the Management and Systems Flight.
Recommend one of the test monitors be individual from the Procedures and Analysis Section and
one from the Computer Operations Section.
2.38.3. 735 SCMG/LGSP test monitors will ensure a meeting is scheduled with the QT&E test
base prior to a major release load to discuss affected program and procedural changes. For
minor release loads, notification via email is sufficient.

2.40. Distribution of Release Documentation. 735 SCMG/LGSP will clearly identify the
programs and procedures in the advance documentation as undergoing testing and forward all
advance documentation to the QT&E test site at the earliest date.
2.42. Evaluation of Testing. The QT&E test base will notify 735 SCMG/LGPS of any program
or procedural problems that occur as a result of the new release load.
2.43. Reporting Program/Procedural Problems. 735 SCMG/LGPS and the QT&E base will
report all test program and procedural problems directly to the SBSS Control Center.
